# Режимы получения событий

Интеграция поддерживает два режима получения входящих событий Telegram.

## WebHook

Telegram вызывает REST-функцию NSD, которую обслуживает `telegramController`.

Этот режим лучше подходит, когда инстанс NSD доступен для запросов Telegram и требуется быстро получать входящие события.

## Long Polling

NSD по расписанию запускает `TelegramUpdateLongPolling`, получает обновления через `getUpdates` и передает их в общий обработчик.

Этот режим удобен, когда входящие запросы от Telegram невозможны или нежелательны.

## Взаимное отключение режимов

Режимы рассчитаны на работу по отдельности:

- `TelegramUpdateWebhook` отключает планировщик Long Polling.
- `TelegramUpdateLongPolling` отключает планировщик WebHook-обновления и удаляет WebHook в Telegram.

Такой подход защищает систему от двойной обработки одних и тех же update.
